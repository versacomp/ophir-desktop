# ophir-desktop

A quantitative trader IDE that combines a Python development environment with AI-powered algorithmic trading. Connects to the tastytrade REST API and DXLink real-time streaming service. Built with PyQt6 and designed to run as a native desktop application.

## Description

ophir-desktop is an open-source desktop IDE built for quantitative developers who want to research, backtest, and deploy algorithmic trading strategies without leaving Python. It streams live 1-minute candle data directly from tastytrade's DXLink WebSocket feed and renders an interactive candlestick chart with signal overlays, indicator sub-panels, and simulated trade management — all inside a single native window with a built-in Python code editor.

The platform is **deliberately strategy-agnostic**. ophir-desktop provides the institutional-grade infrastructure — real-time data, execution engine, performance analytics, and an RL framework — and gets out of the way so you can focus entirely on your alpha. Write your strategy in the IDE, validate it against live market data in simulation, and when it's ready, deploy it to **Ophir Cloud** for low-latency live execution.

ophir-desktop also includes a **Reinforcement Learning** layer (PPO via Stable Baselines3) that can train an agent against a Gymnasium environment built on live market data — giving strategy developers a path from hand-crafted rules to learned policies within the same tool.

**Key design goals:**
- **Bring your own alpha** — the signal, filter, and trade management layers are fully modular; plug in any rules-based or ML strategy with a single Python class
- **Research to production** — develop locally, backtest against cached history, then deploy to Ophir Cloud without changing your strategy code
- Real-time data with graceful demo fallback — the chart is always usable, even without a live API connection
- Sandbox-first — the default environment is tastytrade's certification sandbox; switching to production is an explicit user action
- Transparent simulation — paper trades and back-test results are stored locally and never sent anywhere
- Open and extensible — MIT licensed, modular architecture, no lock-in to a specific broker beyond the tastytrade API client

> ophir-desktop is a port and evolution of the open-source [MTrade](https://github.com/versacomp/mtrade) project, re-architected from Flet to PyQt6 and extended with IDE and AI capabilities.

## Features

- **Python IDE** — built-in code editor with syntax highlighting (Darcula theme via QScintilla), live terminal output, and a background execution engine
- **OAuth2 / sandbox login** — authenticates against the tastytrade sandbox (`api.cert.tastyworks.com`) or production environment
- **Live market dashboard** — real-time quotes for major US indices (S&P 500, Nasdaq-100, Dow Jones) in the toolbar
- **Interactive candlestick chart** — any symbol, 1-minute candles, pan and zoom via mouse/scroll
- **Bring your own alpha** — plug any rules-based or ML strategy into the execution engine; signal overlays, simulation, and analytics adapt automatically
- **Ophir Cloud deployment** — promote a validated strategy from local simulation to low-latency live execution on Ophir Cloud with no code changes
- **Performance dashboard** — KPI metrics, equity curve, and back-test engine over cached history
- **Reinforcement Learning** — Gymnasium environment + PPO agent for training ML-based strategies on live market data
- **DXLink streaming** — WebSocket candle feed with exponential back-off reconnection and transparent demo fallback
- **Persistent candle cache** — 48-hour rolling cache per symbol stored in `~/.ophir-desktop/`

## Quick Start

### Step 1 — Create a tastytrade Sandbox account

The sandbox is a free developer environment that mirrors the production API but resets every 24 hours (positions and balances are cleared; your user account persists).

1. Go to **[developer.tastytrade.com/sandbox](https://developer.tastytrade.com/sandbox/)** and click **"Register here"** to create a sandbox user account. If you already have one, sign in directly.
2. Once logged in, you can create test accounts, submit paper trades, fetch balances and positions, and stream delayed market data — all without risking real capital.

> **Note:** Sandbox quotes are **15 minutes delayed**. Real-time streaming requires a production tastytrade account.

---

### Step 2 — (Optional) Register an OAuth application

Username/password login works out of the box. If you prefer OAuth2 token-based authentication (no password stored):

1. Sign in to your **production** tastytrade account at [my.tastytrade.com](https://my.tastytrade.com/app.html#/manage/api-access/oauth-applications) and navigate to **Manage → API Access → OAuth Applications**.
2. Create a new application — note the **Client ID** and **Client Secret**.
3. Complete the OAuth flow to obtain a **Refresh Token** for your application.
4. Repeat on the sandbox portal for a separate set of sandbox OAuth credentials.
5. Add all values to your `.env` file (see Step 4).

---

### Step 3 — Install dependencies

Requires **Python 3.11+**.

```bash
git clone https://github.com/YOUR_USERNAME/ophir-desktop.git
cd ophir-desktop

python -m venv .venv
.venv\Scripts\activate        # Windows
# or: source .venv/bin/activate  # macOS / Linux

pip install -r requirements.txt
```

---

### Step 4 — Configure credentials

```bash
cp .env.example .env
```

Open `.env` and fill in your credentials:

```ini
# Sandbox — used when Sandbox mode is active
TASTYTRADE_API_BASE_SANDBOX=https://api.cert.tastyworks.com
TASTYTRADE_CLIENT_ID_SANDBOX=your-sandbox-client-id
TASTYTRADE_CLIENT_SECRET_SANDBOX=your-sandbox-client-secret
TASTYTRADE_REFRESH_TOKEN_SANDBOX=your-sandbox-refresh-token

# Production — used when Production mode is active
TASTYTRADE_API_BASE=https://api.tastyworks.com
TASTYTRADE_CLIENT_ID=your-production-client-id
TASTYTRADE_CLIENT_SECRET=your-production-client-secret
TASTYTRADE_REFRESH_TOKEN=your-production-refresh-token
```

OAuth fields are only required if you use OAuth2 token authentication. Username/password login works without them.

---

### Step 5 — Run the app

```bash
python src/main.py
```

The app opens as a native desktop window. Use the **Sandbox / Production** toggle in the toolbar to select your environment (defaults to Sandbox). A status badge in the toolbar confirms the active environment and connection state (`LIVE`, `RECONNECTING`, `OFFLINE`, or `DEMO`).

## Strategy Agnostic

ophir-desktop is built around a single principle: **your edge belongs to you**. The platform provides everything beneath the strategy layer — data, charting, simulation, analytics, and execution — so you can focus entirely on developing alpha.

### The Alpha Engine Interface

Any strategy is expressed as a Python class with a minimal interface. The platform calls your engine on every incoming candle; what you do with it is entirely up to you.

```python
class MyAlphaEngine:
    def on_candle(self, candle: dict, history: pd.DataFrame) -> dict | None:
        """
        Called on every new 1-minute candle.

        Args:
            candle:  Latest OHLCV candle as a dict.
            history: Rolling candle history as a pandas DataFrame.

        Returns:
            A signal dict  {"direction": "long"|"short", "entry": float,
                            "stop": float, "target": float}
            or None for no signal.
        """
        ...
```

The execution engine, simulated trade management, chart overlays, and performance analytics all respond to whatever your engine emits — no strategy-specific assumptions are baked in anywhere.

### Development Workflow

**1. Research in the IDE**

Write and iterate on your strategy directly in the built-in Python editor. The editor has full syntax highlighting, auto-complete with injected market variables (`historical_df`, `latest_tick`, `execute_trade()`), and a live terminal that streams your `print()` output in real time.

**2. Backtest against live history**

The backtester replays your strategy over the locally cached 48-hour candle history with no look-ahead bias — each candle only sees data that existed before it. Results feed directly into the performance dashboard so you can measure your edge before risking capital.

**3. Simulate in real time**

Activate your strategy against the live DXLink feed. Simulated trades are opened, managed, and closed in real time; the blotter and dashboard update tick-by-tick. No real capital at risk — identical behaviour to live, including fills, slippage modeling, and trade management rules.

**4. Deploy to Ophir Cloud**

When your strategy is validated, promote it to **Ophir Cloud** — a low-latency execution platform purpose-built for algorithmic futures trading. Your strategy code is uploaded as-is; Ophir Cloud provides co-located infrastructure, order routing directly to tastytrade's execution layer, and real-time monitoring from the same desktop interface. No code changes required between simulation and live deployment.

### Alerts

When your strategy emits a signal on the active symbol:
- An overlay badge flashes in the top-right corner of the chart
- A snack-bar notification appears at the bottom of the screen
- An audible beep sounds (Windows only, via `winsound`)

## Performance Dashboard

The performance dashboard computes statistics from all persisted simulated trades for the selected symbol and provides a full back-test over the cached 48-hour candle history.

### KPI Metrics

| Metric | Description |
|---|---|
| Win Rate | % of closed trades that were profitable |
| Profit Factor | Gross wins ÷ gross losses |
| Expectancy | Expected pts per trade (win rate × avg win + loss rate × avg loss) |
| Max Drawdown | Largest peak-to-trough equity decline |
| Sharpe | Mean P&L ÷ std dev of P&L per trade |
| Recovery Factor | Total P&L ÷ max drawdown |
| W:L Ratio | Average winning trade ÷ average losing trade |
| Total P&L | Cumulative points across all closed trades |

An equity curve canvas shows the running P&L per trade, colored green/red by sign, with peak and trough markers. A trade breakdown table shows entry source, exit reason, and duration for every closed position.

### Back-test Engine

The back-test replays your strategy over the full cached history with no look-ahead bias — each candle only sees data that existed before it. Strategy parameters can be adjusted and results compared without restarting the data feed.

## Reinforcement Learning

ophir-desktop includes an RL framework for training agents directly on market data.

- **`CitadelEnv`** — a Gymnasium-compatible environment that wraps live or cached candle data; each step provides a 54-dimensional observation vector (OHLCV, technical indicators, account state) and a reward signal based on realized P&L and drawdown penalty
- **`StateVectorizer`** — converts raw market data and account state into a normalized float32 observation suitable for neural network policies
- **`AccountState`** — tracks balance, unrealized P&L, drawdown, and margin usage for reward shaping
- **PPO training** — run via `OphirExecutionEngine` in a background thread so the UI remains responsive during training

To train an agent, write a training script in the IDE editor and execute it using the Run button. The execution engine captures stdout/stderr and streams it to the terminal dock.

## Data Storage

All persistent data lives under `~/.ophir-desktop/`:

```
~/.ophir-desktop/
└── ophir_history.db    # SQLite database: candle cache + closed trade history
```

The candle cache is seeded immediately on symbol activation so the chart shows historical data before the DXLink stream connects. Candle data is stored with a 48-hour rolling window per symbol.

## Streaming Architecture

Market data arrives via **DXLink** — tastytrade's WebSocket-based market data protocol.

**Connection lifecycle:**
1. Fetch a short-lived quote token via `GET /api-quote-tokens`
2. Open WebSocket → `SETUP` → `AUTH` → `CHANNEL_REQUEST` → `FEED_SUBSCRIPTION`
3. Candle events arrive in `COMPACT` format (flattened field arrays)
4. `KEEPALIVE` frames sent every 30 seconds to keep the connection alive

**Resilience:**
- Exponential back-off retry (5s base, 60s ceiling, ±10% jitter, up to 10 attempts)
- A healthy stream that runs for ≥ 60 seconds resets the attempt counter
- After all retries are exhausted, the view automatically falls back to deterministic demo data so the chart remains usable without a live connection
- The connection status indicator in the toolbar reflects `LIVE`, `RECONNECTING`, `OFFLINE`, or `DEMO` in real time

Both the broker and streamer manage dedicated asyncio event loops to prevent conflicts with Qt's own event loop.
